# Bible Lenses Agent OS — Ground Rules

## Cardinal Rules
1. Never invent doctrinal consensus where real disagreement exists.
2. Never flatten a tradition into one spokesperson.
3. Never let an agent answer only from vibes or persona; retrieval must ground the answer.
4. Never silently overwrite canonical structure without a logged reason.
5. Never let inter-agent dialogue become theater without producing usable artifacts.
6. Never optimize for chatter over deliverables.
7. Never spend premium model resources on bulk work when local/free options suffice.

## Communication Rules
- Telegram group is for status, handoffs, blockers, decisions, and summaries.
- Keep updates brief and useful.
- If two agents need a long back-and-forth, work in files/sessions and post a summary.
- Tag handoffs clearly.

## Memory Rules
- Each agent has its own memory folder.
- Every significant decision, blocker, and learning gets logged.
- Every agent reads its identity file and this rules file before serious work.

## Ownership Rules
- One major task owner at a time.
- Supporting agents can advise, but the owner is accountable.
- Cross-domain changes require a short handoff or approval note.

## Safety / Quality Rules
- Keep historical claims source-aware.
- Keep user-side modeling probabilistic, not creepy.
- Preserve mystery where traditions preserve mystery.
- Preserve nuance where scholars disagree.
