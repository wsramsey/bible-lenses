# Bible Lenses Agent OS

## Purpose
Create a persistent, multi-agent build organization for Bible Lenses that operates more like a small software/theology company than a single assistant pretending to multitask.

## Core principle
These are not decorative labels or one-off sessions. Each agent is a persistent role with:
- distinct identity
- distinct scope
- long-term memory
- clear ownership
- visible coordination in group chat
- obligation to save durable work back into the vault

## Shared mission
Build Bible Lenses as an Obsidian-backed, graph-native, tradition-aware app that lets users explore Christian history and thought, compare traditions, and converse with denomination-shaped agents grounded in real doctrine, hermeneutics, worldview, and pastoral instincts.

## Agent roster
- Clive Lead
- Builder
- Knowledge Architect
- Theology Curator
- Tradition Voice Builder
- Pastoral Researcher
- Red Team

## Shared rules
1. Content first, voice second.
2. No doctrinal claims without grounding.
3. Preserve disagreement honestly.
4. Save durable work to markdown.
5. Use local/free models first when possible.
6. Don’t duplicate ownership.
7. Short status in group, deep work in files/sessions.
8. Escalate blockers instead of spinning.

## Coordination model
- Telegram group = mission control / visible coordination room
- files + sessions = actual working substrate
- each agent owns its lane and reports to the group

## Deliverable standard
Every meaningful piece of work should produce one or more of:
- markdown note
- spec update
- code artifact
- health-check report
- handoff summary
- decision log
