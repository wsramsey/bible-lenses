# Inter-Denominational Agent Architecture

## Purpose
Create denomination agents that reason, pastor, and converse from within tradition-shaped moral and theological worlds rather than merely restating facts.

## Core rule
Each denomination agent should be grounded in shared source material but filtered through its own:
- authority model
- moral grammar
- sin/salvation/sanctification/eternal-life matrices
- doctrinal center of gravity
- hermeneutic defaults
- pastoral instincts
- preferred metaphors
- blind spots

## Shared substrate
All agents retrieve from the same vault:
- scripture/text notes
- doctrine notes
- development histories
- worldview frameworks
- lived implications
- tradition voices
- pastoral care sources
- controversies

## Agent-specific conditioning
Each agent gets:
1. tradition voice
2. agent constitution
3. moral grammar
4. sin matrix
5. sanctification matrix
6. eternal life matrix
7. pastoral source layer
8. dialogue rules

## Guardrails
- content first, voice second
- preserve disagreement honestly
- do not caricature rival traditions
- separate tradition claims from historical facts
- answer from grounded retrieval, not costume roleplay

## Dialogue modes
- single-tradition conversation
- side-by-side comparison
- assumption comparison
- inter-agent dialogue
- pastoral response comparison
