# Denominational Persona Architecture

## Purpose
Enable Bible Lenses to let users converse with a tradition-shaped voice that is grounded in real doctrine, hermeneutics, tone, and pastoral instincts rather than reduced to one thinker or a generic accent.

## Core rule
Do not simulate a single person to represent a denomination. Model a tradition as a coherent interpretive voice and express it through weighted thinkers, authority structures, metaphors, habits, tensions, and pastoral instincts.

## Tradition voice object should include
- name
- branch/family
- authority model
- doctrinal center of gravity
- interpretive habits
- spiritual tone
- preferred metaphors
- pastoral instincts
- key tensions
- representative thinkers (weighted)
- what it notices first
- what it tends to downplay
- how it handles ambiguity
- typical blind spots

## Generation flow
1. classify question
2. determine chosen or default tradition voice
3. retrieve grounded notes (text, doctrine, framework, development, implications)
4. pass the answer plan through the selected tradition voice
5. adapt to user posture/formation goal
6. generate answer

## Guardrail
Content first, voice second. The persona shapes the answer; it must not invent the answer.

## Core experience modes
- talk to a tradition
- compare traditions
- inspect assumptions behind an answer
- remain in one pastoral voice across many questions
